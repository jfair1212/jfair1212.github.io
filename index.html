Enhancement One: Calendar View

Original State: Initially, the main screen of the application included a RecyclerView that held a scrolling list of each event scheduled by the user, sorted by date. Each event showed in the form of a card that held all the event's information as well as options to edit or delete each event.

Enhancement: To increase user-friendliness, I replaced the scrolling list with a CalendarView. The database was queried and each date with a scheduled event was given a visual marker through the use of the Calendar external library by Kizitonwose. Each day can be clicked on, which will take the user to a screen that displays that day's events in a similar event card fashion. From there, the event information and edit/delete options are open to the user as before.

Skills Displayed: This enhancement displayed my ability to refactor my project from a RecyclerView scrolling list to a CalendarView while still maintaining functionality. It also displayed my ability to integrate an external library to acheive my project goals. This section satisfied Course Outcome 3 through designing a solution to a user problem and Course Outcome 4 through the use of industry-standard tools to deliver value.

Enhancement Two: Search Feature

Original State: Prior to this feature, the user had to manually scroll through their calendar events to find a specific one. There was no alternative method to filtering and searching events. As the user's event list grew, this became increasingly inefficient. Searching for specific events could become time consuming and negatively impact overall usability.

Enhancement: I implemented a dedicated search screen with a search bar at the top. Each keystroke in the search bar sends a query to the database, searching for any events that include the user's query in its title or description. Each matching event is presented in the same event card fashion inside of a scrolling list. If no matching events are found, a corresponding message is displayed to notify the user.

Skills Displayed: This section displays my ability to provide new functionality through XML and Java. I showed that I can handle database querying for specific text while maintaining a dynamic and responsive UI to increase user satisfaction and efficiency. This enhancement satisfies Course Outcome 3 through solving a user porblem through algorithmic thinking and problem solving.

Enhancement Three: Firebase

Original State: The application initially relied on a locally stored SQLite database. This satisfied the initial project requirements but limited any user's access to their account to just one device which is unrealistic for consumer use. The database schema originally held a table for users and one for events. The users table held user credentials while the events table held all other information for the events with each event linked to a user by a user_id.

Enhancement: I replaced the local SQLite database with Firebase. Specifically, I used Firebase Authentication to handle user log-in/sign up and Firestore for the event storage. Full functionality of the application was retained through this switch. This enables cross-device access of a user's account as well as increased security through Firebase Authentication and Firestore.

Skills Displayed: I displayed my ability to refactor my application to integrate a new database while without affecting performance. Additionally, I increased the application's sceurity measures through Authentication's built-in hashing and user validation. Alongside that, Firestore has built in data security measures as well. Due to this, Course Outcome 5 was satisfied by this enhancement. Additionally, Course Outcome 4 was satisfied through the use of Firebase to improve the functionality of the application.
